# 实验操作指导

提前掌握实验关键缓存概念，避免实现时踩坑：

1. 缓存地址划分：64 位地址拆分为**标记位（tag）、组索引位（s 位）、块偏移位（b 位）**，组数量`S=2^s`，块大小`B=2^b`；
2. LRU 替换策略：当缓存组满时，替换**最近最少使用**的缓存行；
3. 内存操作类型：忽略`I`（指令加载），处理`L`（数据加载）、`S`（数据存储）、`M`（数据修改，=L+S）；
4. 直接映射缓存：`E=1`，每个组仅有 1 个缓存行，易产生冲突未命中；
5. 分块（Blocking）技术：将大矩阵划分为小分块，让分块能完全放入缓存，减少缓存行的频繁替换。

# A部分：编写缓存模拟器

在 A 部分，你需要在 csim.c 文件中实现一个缓存模拟器。该模拟器接收 valgrind 生成的内存跟踪信息作为输入，模拟缓存对该跟踪信息的命中 / 未命中行为，并输出总命中数、未命中数和替换数。

我们已为你提供了参考缓存模拟器的二进制可执行文件 csim-ref，该程序可模拟任意大小、任意相联度的缓存对 valgrind 跟踪文件的处理行为，选择替换的缓存行时采用**最近最少使用（LRU）** 替换策略。

参考模拟器的命令行参数格式如下：

```
用法：./csim-ref [-hv] -s <s> -E <E> -b <b> -t <tracefile>
```

参数说明：

- `-h`：可选的帮助标志，打印使用说明
- `-v`：可选的详细输出标志，显示跟踪信息
- `-s <s>`：组索引位的位数（缓存组数量 S = 2^s）
- `-E <E>`：相联度（每个缓存组中的缓存行数量）
- `-b <b>`：块偏移位的位数（缓存块大小 B = 2^b）
- `-t <tracefile>`：待重放的 valgrind 跟踪文件名称

上述命令行参数的符号（s、E、b）参考《深入理解计算机系统（第二版）》教材 597 页的定义。示例：

```
linux> ./csim-ref -s 4 -E 1 -b 4 -t traces/yi.trace
```

你的任务是完善 csim.c 文件，使其接收与参考模拟器相同的命令行参数，并输出完全一致的结果。请注意，该文件初始状态几乎为空，你需要从头编写。

#### A 部分编程规则

1. 在 csim.c 的头部注释中注明你的姓名和登录 ID；

2. 你的 csim.c 文件必须能无警告编译通过，否则不予计分；

3. 模拟器必须能针对任意的 s、E、b 正确运行，这意味着你需要使用 malloc 函数为模拟器的数据结构动态分配内存，可通过执行`man malloc`查看该函数的使用说明；

4. 本实验仅关注**数据缓存**的性能，因此模拟器需忽略所有指令缓存的访问（以`I`开头的行）。注意，valgrind 生成的跟踪信息中，`I`始终在第一列（无前导空格），`M`、`L`、`S`始终在第二列（有前导空格），这一特点可帮助你解析跟踪文件；

5. 要获得 A 部分的分数，你必须在 main 函数末尾调用 printSummary 函数，传入总命中数、未命中数和替换数：

   ```
   printSummary(hit_count, miss_count, eviction_count);
   ```

6. 本实验假定内存访问是**对齐**的，单次内存访问绝不会跨缓存块边界。基于此假定，你可以忽略 valgrind 跟踪信息中的访问字节数。



# B部分：优化矩阵转置函数

在 B 部分，你需要在 trans.c 文件中编写一个矩阵转置函数，尽可能减少缓存未命中的次数。

设矩阵为 A，A_ij 表示其第 i 行第 j 列的元素。A 的转置矩阵记为 A^T，满足 A_ij = A^T_ji。

为帮助你上手，我们在 trans.c 中提供了一个示例转置函数，可计算 N×M 矩阵 A 的转置，并将结果存储到 M×N 矩阵 B 中：

```
char trans_desc[] = "Simple row-wise scan transpose";
void trans(int M, int N, int A[N][M], int B[M][N])
```

该示例函数逻辑正确，但效率低下，因其内存访问模式会导致较多的缓存未命中。

你在 B 部分的任务是编写一个类似的函数 transpose_submit，使该函数在不同大小的矩阵上都能最小化缓存未命中次数：

```
char transpose_submit_desc[] = "Transpose submission";
void transpose_submit(int M, int N, int A[N][M], int B[M][N]);
```

**请勿修改**transpose_submit 函数的描述字符串（"Transpose submission"），自动评分程序会通过该字符串识别用于计分的转置函数。

#### B 部分编程规则

1. 在 trans.c 的头部注释中注明你的姓名和登录 ID；
2. 你在 trans.c 中的代码必须能无警告编译通过，否则不予计分；
3. 每个转置函数最多只能定义**12 个 int 类型的局部变量**；
4. 不允许通过使用 long 类型变量、或利用位操作技巧将多个值存储到单个变量中，规避上述规则；
5. 转置函数不得使用递归；
6. 若选择使用辅助函数，辅助函数与顶层转置函数的栈上局部变量总数**单次不得超过 12 个**。例如，若转置函数声明了 8 个变量，调用的第一个辅助函数使用 4 个变量，该辅助函数又调用了一个使用 2 个变量的函数，此时栈上共有 14 个变量，违反本规则；
7. 转置函数**不得修改**数组 A 的内容，但可对数组 B 的内容进行任意操作；
8. **不允许**在代码中定义任何数组，也不允许使用任何形式的 malloc 函数。



# PartA 缓存模拟器实现

### 实验任务

填充`csim.c`，实现一个与`csim-ref`行为完全一致的缓存模拟器，支持任意`s/E/b`参数，解析跟踪文件并输出**总命中数、未命中数、替换数**，最终调用`printSummary(hit, miss, evict)`输出结果。

### 实现整体流程

```
命令行参数解析 → 缓存数据结构初始化 → 跟踪文件逐行解析 → 内存访问模拟（命中/未命中/LRU替换） → 统计结果 → 调用printSummary → 释放内存
```

### 关键实现步骤（核心思路）

#### 步骤 1：命令行参数解析（使用 getopt）

- 需解析的参数：`-s <s> -E <E> -b <b> -t <tracefile>`，可选`-h`（帮助）、`-v`（详细模式，建议实现）；
- 头文件依赖：`#include <getopt.h> <stdlib.h> <unistd.h>`；
- 关键：对参数做合法性校验（s/E/b 为非负整数，tracefile 存在）。

#### 步骤 2：设计缓存数据结构（动态分配，malloc）

模拟器的核心是模拟缓存的物理结构，需为**缓存组**和**缓存行**设计结构体，支持 LRU 标记：

1. **缓存行（Cache Line）** 结构体：包含**标记位（tag）**、**有效位（valid，标记行是否有效）**、**LRU 时间戳（lru_cnt，标记最近使用时间，值越大越近）**；
2. **缓存**：二维结构（数组），`cache[S][E]`，`S`为组数量，`E`为每个组的行数量，需用`malloc`动态分配（支持任意`s/E/b`）；

- 关键：计算`S=1<<s`（2^s），注意整数类型（用`unsigned long`存储地址 /tag，避免溢出）。

#### 步骤 3：跟踪文件解析

1. 逐行读取`-t`指定的跟踪文件，**忽略以`I`开头的行**（指令加载）；
2. 解析每行的操作类型（op）、内存地址（addr）、访问大小（size，可忽略，实验假定地址对齐）；
   - 行格式：`[空格]op addr,size`（op 为 L/S/M）；
   - 地址转换：将十六进制字符串 addr 转为 64 位无符号整数（`strtoul`函数）；
3. 提取地址的组索引（set_idx）**和**标记位（tag）：
   - 组索引：`(addr >> b) & (S - 1)`（右移 b 位屏蔽块偏移，与 S-1 相与取 s 位）；
   - 标记位：`addr >> (s + b)`（右移 s+b 位，取高位所有位）。

#### 步骤 4：缓存访问模拟（核心逻辑）

对每个数据操作（L/S/M），执行缓存查找逻辑，更新命中 / 未命中 / 替换数，**M 操作需执行两次查找（先 L 后 S）**：

##### 通用查找逻辑（单步 L/S）

1. 命中判断：在对应组cache[set_idx]的 E 行中，查找有效位 = 1 且 tag 匹配的行；
   - 若命中：更新该行为**最新 LRU**（lru_cnt 自增并赋值），命中数`hit_count++`，结束；
   - 若未命中：未命中数`miss_count++`，进入步骤 2；
2. 空行判断：在该组中查找有效位 = 0的空行；
   - 若有空行：将当前 tag 写入该空行，置有效位 = 1，更新 LRU 时间戳，结束；
   - 若无空行：替换数`evict_count++`，进入步骤 3；
3. **LRU 替换**：在该组中查找**LRU 时间戳最小**的行，用当前 tag 覆盖其 tag，更新 LRU 时间戳，结束。

- 关键：LRU 时间戳用全局计数器（如`unsigned int lru_clock`），每次访问缓存时自增，赋值给当前行，保证 “最近使用” 的行戳最大。

#### 步骤 5：内存释放与结果输出

1. 模拟完成后，用`free`释放`malloc`分配的缓存内存（避免内存泄漏）；
2. 严格调用实验提供的`printSummary(hit_count, miss_count, eviction_count)`输出结果，**不可自定义输出格式**（否则 test-csim 无法识别）。

### 编程规则严格遵守

1. `csim.c`头部注释注明姓名 / 登录 ID；
2. 代码无警告编译（GCC 编译默认开启`-Wall`，需消除所有警告）；
3. 仅关注数据缓存，忽略所有`I`开头的指令加载行；
4. 动态分配内存，支持任意`s/E/b`参数，不可硬编码；
5. 忽略跟踪文件中的`size`字段，假定地址对齐。
